# 별 컨셉 모바일 청첩장 프로젝트

## 프로젝트 개요
- **무엇**: 템플릿 기반 모바일 청첩장 부업 (셀프 서비스 + 디렉터 검수)
- **포지션**: 합법 프리미엄 (정식 라이선스 음원 + 디렉터 큐레이션)
- **타겟**: 20-30대 신랑신부
- **핵심 차별점**: 8번 섹션의 "별 인터랙션" (축하 메시지가 별로 쌓여서 하트 모양 별자리가 됨)

전체 명세서는 `docs/spec.html` 참고. 작업 전에 항상 관련 섹션 먼저 확인.

## 기술 스택
- **프론트엔드**: Next.js (App Router) + TypeScript + Tailwind CSS
- **백엔드/DB**: Supabase (나중에 도입, 지금은 정적 페이지부터)
- **호스팅**: Vercel
- **결제**: 초기 계좌이체 → 토스페이먼츠

## 코딩 규칙

### 작업 방식
- **한 번에 한 단계만**. 큰 변경 금지. 욕심내지 말 것.
- 파일 만들거나 수정하기 전에 **무엇을 할지 먼저 한국어로 설명**할 것.
- 명세서에 없는 기능은 임의로 추가하지 말 것. 필요하면 먼저 물어볼 것.
- 모르는 게 있거나 명세서에 모호한 부분이 있으면 **추측하지 말고 질문**할 것.

### 코드 스타일
- TypeScript 엄격 모드 유지
- 컴포넌트는 `components/` 폴더에 기능별로 분리
- 색상·폰트·여백은 **반드시 Tailwind config의 토큰 사용**. 하드코딩 금지.
- 한글 주석 환영 (사용자 본인 부업이므로 본인이 읽기 편한 게 우선)

### 모바일 우선
- 청첩장은 **거의 100% 모바일에서 열림**. 모바일 뷰포트 기준으로 먼저 디자인.
- 기본 폭은 `max-w-[420px]` 정도 잡고, 데스크탑에서는 가운데 정렬.

## 절대 금지사항 (저작권 · 음원 정책)

이 프로젝트의 핵심 정체성은 **합법성**. 다음은 절대 안 됨:
- K-pop, 가요 음원 사용 (명세서 14번)
- 고객 mp3 직접 업로드 기능 (명세서 14번)
- 특정 작가·브랜드 스타일 모방 AI 이미지 (명세서 21번)
- 라이선스 불확실한 폰트·이미지·음원 사용
